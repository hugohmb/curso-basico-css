Em projetos web, é possível aplicar estilos (cores, fontes, tamanhos etc.) de diferentes maneiras, e cada uma delas possui uma prioridade diferente. Quando mais de um estilo tenta formatar o mesmo elemento, o navegador segue uma ordem de precedência para decidir qual estilo aplicar.

O uso de um arquivo CSS externo (com extensão .css) é bastante comum, pois permite reutilizar os mesmos estilos em várias páginas do site, facilitando a padronização e a manutenção do visual. No entanto, se houver estilos definidos também no próprio arquivo HTML, eles podem sobrescrever o que está no CSS externo.

Essa prioridade funciona da seguinte forma:

1 - Estilo direto na tag HTML (chamado de inline style) tem a maior prioridade. Exemplo: <h1> style="color: red;">Título</h1>


2 - Estilo definido dentro da tag <style> no <head> do HTML tem prioridade intermediária. Exemplo: 


    <style>
        h1 {
          color: blue;
        }
        </style>

3 - Estilo vindo de um arquivo CSS externo tem a menor prioridade, sendo aplicado apenas se nenhum dos anteriores estiver interferindo. Exemplo: style.css contendo:


    h1 {
        color: green;
      }

Apesar dessa hierarquia, os estilos podem ser combinados. Por exemplo, a cor de um elemento pode ser definida pelo estilo inline, enquanto o tamanho da fonte pode vir do CSS externo — desde que essas regras não entrem em conflito (ou seja, não tentem modificar a mesma propriedade).

Essa flexibilidade permite um controle detalhado da aparência, mas exige atenção para evitar confusão ou conflitos desnecessários.
